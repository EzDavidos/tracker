# Habits Creator (tracker)

Веб-приложение для ежедневного трекинга привычек с системой очков и наградами.
Статус: in progress, неполное.

## Стек

- Plain HTML/CSS/JS, **всё в одном файле** `index.html` (~1100 строк)
- Firebase Firestore через CDN (modular SDK v10.12) — динамический import внутри inline-скрипта
- localStorage как fallback (`habits_creator_v1`)
- Деплой: GitHub Pages
- UI на русском

## Архитектура

Состояние в двух глобальных объектах:
- `db` — персистентное (месячный счёт, история, недели, конфиг блоков). Сохраняется в Firestore + localStorage
- `day` — состояние текущего просматриваемого дня (`dayOffset` смещает дату)

Поток данных: действие пользователя → мутация `day`/`db` → `localSave()` + `cloudSave()` → `updateScores()` + `renderBlocks()`.

### Типы блоков (`DEFAULT_BLOCKS`)
1. `sport` — выбор варианта тренировки + недельный бонус за target дней
2. `cs` — таймер с лимитом минут, очки зависят от того, уложился ли в лимит
3. `weed` — clean/dirty за будний день
4. `multi` — мульти-выбор (вечерняя активность)

При добавлении/правке блока смотреть **все четыре** рендерера, иначе один из типов сломается.

## Правила работы

- Не разбивать `index.html` на модули без явной просьбы — текущая структура осознанная (работает и в Claude artifact, и на GitHub Pages без сборки)
- Не рефакторить то, что не просили
- При правке формата `db` (поля, структура блоков) учитывать **миграцию существующих данных** в Firestore у пользователя — добавлять дефолты при чтении, а не падать
- UI и тексты — русские, сохранять стиль
- Тестировать вручную в браузере: основной поток (отметить день → счёт → история) + переключение `dayOffset`
